### Task 3: Initialize Project Structure (45 minutes)

**What to do:**
Create the Konfiguru project directory structure following Go best practices.

**Step-by-step:**

1. **Create project root and initialize module:**
```bash
cd ~/Documents/Side-Projects/konfiguru
go mod init github.com/bhargav/konfiguru
```

Expected output: `go: creating new go.mod: module github.com/bhargav/konfiguru`

2. **Create directory structure:**
```bash
mkdir -p cmd/lox
mkdir -p pkg/tokens
mkdir -p pkg/lexer
mkdir -p test/fixtures
mkdir -p examples
mkdir -p docs
```

**Directory purpose:**
- `cmd/lox/` - Main application entry point (REPL)
- `pkg/tokens/` - Token types and definitions
- `pkg/lexer/` - Scanner/lexer implementation
- `test/fixtures/` - Test input files
- `examples/` - Example Lox programs
- `docs/` - Additional documentation

3. **Create .gitignore:**
```bash
cat > .gitignore << 'EOF'
# Binaries
*.exe
*.exe~
*.dll
*.so
*.dylib
lox
konfiguru

# Test binary
*.test

# Output
*.out

# Go workspace file
go.work

# IDE
.vscode/
.idea/
*.swp
*.swo

# OS
.DS_Store
EOF
```

4. **Create Makefile:**
```bash
cat > Makefile << 'EOF'
.PHONY: build test run clean

build:
	go build -o lox ./cmd/lox

test:
	go test -v ./...

run:
	go run ./cmd/lox

clean:
	rm -f lox
	go clean
EOF
```

5. **Verify structure:**
```bash
tree -L 2 .
```

## ðŸ“ Learning Notes

**What is a lexer?**
A lexer (or scanner) is the first phase of a compiler. It reads source code (text) and breaks it into tokens (meaningful chunks like keywords, operators, numbers).

**Example:**
```
Input:  var x = 5;
Tokens: VAR, IDENTIFIER(x), EQUAL, NUMBER(5), SEMICOLON
```

**Why Go for this project?**
- Simple, fast compilation
- Great standard library for text processing
- Static typing helps catch bugs
- Good for learning system programming

**Tomorrow's preview:**
You'll define all the token types Lox needs (keywords, operators, literals) and write tests for them.
