**Create Git workflow doc:** `docs/git-workflow.md`

```markdown
# Git Workflow for Konfiguru

## Branch Strategy

**Simple approach (recommended for learning):**
- Work on `master` branch
- Make frequent, small commits
- Each commit should be working code
- Use conventional commit messages

## Commit Message Format

```
<type>: <description>

[optional body]
```

**Types:**
- `feat:` New feature
- `fix:` Bug fix
- `test:` Add/update tests
- `refactor:` Code refactoring
- `docs:` Documentation
- `chore:` Maintenance

**Examples:**
```
feat: add Binary expression AST node
test: add tests for expression parsing
refactor: simplify parser error handling
docs: document visitor pattern usage
```

## Daily Workflow

1. Start day: Review yesterday's commit
2. Make changes: Small, focused
3. Test: Ensure tests pass
4. Commit: Clear message
5. End day: Push to GitHub

## When to Commit

- After completing a task
- When all tests pass
- Before taking a break
- End of coding session

**Commit often!** Small commits are easier to understand and debug.
```
