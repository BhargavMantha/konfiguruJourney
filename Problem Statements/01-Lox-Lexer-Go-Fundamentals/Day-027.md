## üí° Key Learning Points

### Lexer vs Parser

**Lexer (Month 1):**
- Input: String of characters
- Output: Stream of tokens
- Answer: "What are the words?"

**Parser (Month 2):**
- Input: Stream of tokens
- Output: Abstract Syntax Tree
- Answers: "What is the structure/meaning?"

### Why ASTs Matter

An AST is the bridge between syntax (what the code looks like) and semantics (what it means).

```
Code: "1 + 2 * 3"
       ‚Üì Lexer
Tokens: [NUMBER 1] [PLUS] [NUMBER 2] [STAR] [NUMBER 3]
       ‚Üì Parser
AST:       +
          / \
         1   *
            / \
           2   3
       ‚Üì Interpreter
Result: 7
```

### Grammar and Precedence

**Grammar** defines what is valid syntax.
**Precedence** defines what happens first.

Example: `2 + 3 * 4`
- Grammar says: Both + and * are valid
- Precedence says: * before +
- Result: `2 + (3 * 4) = 14`
